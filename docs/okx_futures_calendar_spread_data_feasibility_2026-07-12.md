# OKX 交割合约跨期价差数据可行性审查

**审查日期**：2026-07-12  
**审查范围**：OKX 官方免费交割合约（Futures）历史 K 线数据  
**审查目的**：评估该数据在不引入付费源的前提下，能否满足 365 天连续验证，并识别潜在的研究与执行风险。  
**仅新建**：`docs/okx_futures_calendar_spread_data_feasibility_2026-07-12.md`

## 一、数据获取与复现性评估

### 1. 官方 API 是否免费、公开、可复现？
**结论**：**是**。
- **获取通道**：OKX 提供官方公开历史归档接口 `GET https://openapi.okx.com/api/v5/public/market-data-history`。
- **参数说明**：设置 `module=2`（K线模块）、`instType=FUTURES`、`dateAggrType=monthly`（按月归档）、`instFamilyList=BTC-USDT` 等 base 资产。
- **验证结论**：该接口完全免费开放，无需 API 秘钥或账户鉴权，下载的文件为 ZIP 压缩包，解压后为标准 CSV 格式 K 线数据，数据源 100% 可由任何研究者复现。

---

## 二、连续覆盖与对齐可行性

### 2. 是否能连续覆盖至少 365 天？
**结论**：**是，但必须通过跨合约拼接（Stitching/Rollover）实现**。
- **交割合约生命周期**：单个交割合约有固定寿命（当周/次周通常为 7-14 天，当季度/次季度通常为 3-6 个月）。因此，没有任何单一的交割合约代码能够直接提供 365 天的连续数据。
- **拼接方案**：为进行 365 天的策略回测，必须下载该周期内所有已过期交割合约的历史归档（例如：季度合约 `BTC-USDT-240927`、`BTC-USDT-241227`、`BTC-USDT-250328`、`BTC-USDT-250627` 等），并在代码中设计明确的 **换月规则（Rollover Schedule）**（如：在合约交割前 N 天，或当新合约成交量超过旧合约时，切换主力合约数据源）。

### 3. 是否能与 OKX 永续/现货对齐？
**结论**：**是**。
- **对齐基准**：所有交割合约的归档 K 线时间戳（`open_time`/`timestamp_ms`）均为标准 UTC 时间（Unix Epoch 毫秒）。
- **对齐可行性**：可与现货（SPOT） and 永续（SWAP）归档 1m K 线数据按 timestamp 进行一一映射和对齐，从而计算出精确的“交割-永续价差（Calendar Spread）”或“交割-现货价差”。

## 三、策略研究与执行风险

在进入事件审计前，必须在研究卡中明确以下六大风险特征：

### 1. 换月价格断裂与数据调整风险（Rollover Gap）
- **现象**：在切换主力合约时（例如从当季切换至次季），由于到期时间不同，两者存在天然的价差跃迁。
- **风险**：如果在回测中直接使用未调整的拼接价格，会导致指标（如均线、通道、RSI）在换月点产生假突破信号。必须使用**价差绝对值**进行研究，或对历史价格进行前向/后向复权调整（Forward/Backward Adjustment），这会增加系统复杂性并影响绝对价格指标的有效性。

### 2. 换月执行摩擦成本（Rollover Cost）
- **现象**：持仓跨期策略在合约临近到期时，必须平仓旧合约并开仓新合约。
- **风险**：每次换月操作都是一次完整的买卖交易，会产生两次手续费和双重滑点。对于高频或中频换月（如使用周度合约），换月损耗会严重侵蚀策略利润。

### 3. 双腿交易与多倍摩擦成本（Multi-Leg Friction）
- **现象**：价差套利或跨期交易通常需要同时操作两个合约（例如：买入永续，卖出季度交割）。
- **风险**：
  - **往返共 4 腿交易**：建仓 2 腿（Perp Buy + Future Sell） + 平仓 2 腿（Perp Sell + Future Buy）。
  - **摩擦成本翻倍**：单腿往返成本若为 `0.16%`（Taker 0.05% + 滑点 0.03%），双腿跨期往返的真实摩擦成本将达到 **至少 0.32%**。策略的原始价差波动必须显著大于 0.32% 才具备可交易性，这过滤掉了绝大多数微观价差机会。

### 4. 单腿成交风险与执行滞后（Leg Risk）
- **现象**：由于 OKX 不支持官方的“跨期组合单（Spread Order）”原子执行，策略必须在客户端发送两条独立的限价单/市价单。
- **风险**：在市场剧烈波动时，可能出现一腿成交而另一腿未成交（Legging Out）的情况，使策略被迫暴露在单向敞口（Naked Position）中。此外，执行延迟会导致价差在我们发送订单的瞬间发生转移，产生严重的负向执行滑点。

### 5. 流动性萎缩与交割前异常波动（Liquidity Collapse）
- **现象**：交割合约在临近交割的最后一周，成交量和持仓量会急剧萎缩，做市商撤单，买卖价差（Bid-Ask Spread）异常变宽。
- **风险**：在流动性极差的时段平仓或换月会遭遇极大的惩罚性滑点。若拖延至交割，OKX 将收取强制交割手续费（通常为 `0.05%` 甚至更高，依账户等级而定），这远高于主动平仓的 Maker 费率。

### 6. 合约代码与到期规则的维护成本
- **现象**：OKX 每周/每季都会上线新合约并下线过期合约，合约代码（如 `BTC-USDT-YYMMDD`）是动态变化的。
- **风险**：代码中必须维护一个完备的合约字典与生命周期状态机，自动识别当前可交易的合约代码，若状态机出错（例如向已下线合约发单），将直接导致系统崩溃。

## 四、审查结论

### **结论：可研究（Eligible for Research）**

- **判定理由**：OKX 官方 `market-data-history` 接口确实提供了完整、免费、公开的交割合约历史归档数据，能够支持 365 天以上的连续拼接与对齐。
- **先决条件**：
  1. 必须编写专门的 expired contracts 数据下载与**拼接逻辑**，处理好换月跃迁。
  2. 回测摩擦成本配置**必须硬性设为双边 4 腿合计至少 0.32%**。
  3. 回测必须加入换月平仓摩擦，并在交割前至少 3 天强制完成换月，避开流动性枯竭期。
  4. 必须通过预注册研究卡声明其持仓逻辑，方可进入事件审计。
