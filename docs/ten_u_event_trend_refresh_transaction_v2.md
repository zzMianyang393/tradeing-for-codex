# v2 前瞻刷新异常事务回滚

一次小时刷新会修改三个币的 K 线、资金费率、工作 manifest、信号账本、刷新报告和哈希审计链。
如果普通程序异常发生在这些写入之间，部分更新会破坏跨文件指纹连续性并造成后续观察停摆。

现在刷新入口会在开始前保存所有可能修改文件的原始字节。只有整个刷新函数成功返回才保留新状态；
任意网络、解析、校验或文件写入异常都会：

1. 将原有文件逐字节恢复；
2. 删除本轮异常期间新建的文件；
3. 重新抛出原始异常，使自动任务明确失败；
4. 下一次调度从上一次完整审计状态继续。

这提供的是“程序异常原子性”，不宣称可以抵御进程被强杀、操作系统崩溃或机器断电。后者仍需要后续
写前日志或原子目录切换。修正不接触策略、信号结果或前瞻收益。
