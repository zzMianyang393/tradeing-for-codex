# UTC 时段突破：预注册研究卡

## 家族定义

本研究卡统一代表以下同一机制家族的策略：

- 开盘区间突破（Opening Range Breakout, ORB）
- 箱体突破（Box Breakout）
- 时间切片突破（Time-Slice Breakout）
- Funding 结算后窗口突破
- 亚盘/欧盘/美盘开盘突破

**核心机制：** 在预定义的 UTC 时段内形成价格区间，区间完成后向突破方向入场。

**选定代表规则：** UTC 00:00–04:00 亚盘区间突破。

## 预注册参数窗口

| 参数 | 值 | 说明 |
|---|---|---|
| 区间时段 | UTC 00:00–04:00（4 小时） | 亚盘开盘区间 |
| 区间定义 | 时段内最高价和最低价 | 不使用中位数或其他 |
| 突破确认 | 区间结束后，某根已完成 15m K 线收盘价高于区间高点 | 仅做多，不用影线穿透 |
| 入场 | 突破确认后的下一根 15m 开盘 | UTC 04:15 只是最早可能入场时间 |
| 止损 | 2.0 × ATR(14)，基于区间内波动 | 固定止损 |
| 止盈 | 3.0 × ATR(14) | 风险回报比 1.5:1 |
| 持有期上限 | 80 根 15m（20 小时） | 当日收盘前强制平仓 |
| 方向 | 仅做多突破（区间高点突破） | 不做空 |

**不得扫描的参数：** 区间时段（00-04/02-06/08-12/14-16）、区间长度（1h/2h/4h）、
止损 ATR 倍数（1/1.5/2/3）、止盈倍数、持有期、方向（做多/做空/双向）、
币种集合。

## 数据

- OKX 15m OHLCV（28 币种，年度覆盖）
- 成本：双边 taker fee 0.05% + 滑点 0.03%，往返 **0.16%**
- 不使用 tick 数据、盘口数据或逐笔成交数据
- 不使用 funding 数据（不与 funding 结算窗口耦合）

## 形成期 / OOS 分割

| 阶段 | 范围 | 用途 |
|---|---|---|
| 形成期 | 2024-01-01 至 2024-12-31 | 事件审计 + 通过门槛 |
| 样本外 | 2025-01-01 至 2025-07-10 | 冻结规则后验证 |

## 通过门槛（形成期）

1. 至少 60 个去重交易事件（跨币种合计，仅做多突破）；
2. 扣成本后净收益均值为正；
3. 净胜率 >= 45%（突破策略通常胜率较低，但盈亏比高）；
4. 盈利因子（总盈利 / 总亏损）>= 1.2；
5. 不由单一币种贡献超过 30% 的总收益；
6. 不由单一月份贡献超过 25% 的事件。

## 样本外验证（形成期通过后）

1. 样本外净收益均值必须为正；
2. 样本外盈利因子 >= 1.0；
3. 样本外胜率 >= 40%。

## 失败后不得做的事

- 不改 UTC 时段（00-04 变 02-06 或 08-12）
- 不改区间长度（4h 变 2h 或 6h）
- 不改止损/止盈 ATR 倍数
- 不改持有期
- 不加过滤条件（成交量、RSI、趋势过滤等）
- 不缩币种集合（仅 BTC、仅高流动性等）
- 不改方向（加做空或改为双向）
- 不改突破确认方式（用影线、用百分比等）
- 不与 funding 结算窗口耦合

## 适用行情

本策略理论上适用于**趋势行情或波动率扩张行情**。
震荡行情中会出现频繁假突破导致亏损。
可使用 `regime_validation.py` 的行情标签在事后审计中拆分表现，
但**不得**在形成期用行情标签过滤信号（这等同于事后选择）。

## 与其他方向的隔离

- 与已淘汰的 `volatility_compression_breakout` 不同：后者使用 ATR 压缩 + 成交量
- 与已淘汰的 `low_turnover_trend` 不同：后者使用换手率 + 4h Donchian
- 与 C2（唐奇安+ATR）不同：C2 使用 N 日高低点突破，本方向使用日内时段区间突破
- 本方向是纯时段区间 + ATR 止损，不含成交量、换手率或趋势过滤

## 研究流程

1. 先写独立审计脚本（不接入 `candidate_strategies.py`）
2. 输出形成期事件报告
3. 通过门槛后，冻结规则
4. 运行样本外验证
5. 全部通过后，才可接入 `unified_validation.py`

## 可证伪性

- 若亚盘区间突破后价格不延续（均值回复为主），假设不成立
- 若假突破率过高（胜率 < 30%），假设不适用于当前市场
- 若成本吞噬大部分利润，假设不可交易
- 若"时段效应"不存在（不同 UTC 时段无显著差异），假设无机制基础

## 禁止接入

- 不接入 `runner.py`
- 不接入 `candidate_strategies.py`
- 不接入模拟盘或实盘
- 形成期通过前不接入 `unified_validation.py`
