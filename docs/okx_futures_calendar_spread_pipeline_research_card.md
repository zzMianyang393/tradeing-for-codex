## 一、研究假设与设计背景

交割与永续合约（或不同交割期合约）之间存在由于到期时间差、基差溢价和资金费率预期导致的合理价差变动。当价差偏离历史均值达到极端分位数时，可能存在均值回复机会。

为了在满足年度门槛（365天）的无偏回测中检验该机制，首先需要建立一个高度严密、排除前视偏差的**交割-永续/交割-交割数据管线**。本文件为该数据管线与拼接逻辑的预注册研究卡，**不涉及收益策略的编写**。

---

## 二、数据管线设计要求

### 1. OKX 官方历史归档数据下载范围
为保证完全免费、公开、可复现，且同源于 OKX 市场，数据管线必须支持下载并校验以下三类数据（格式为 `market-data-history` 接口的月度 ZIP 归档，`module=2`）：
- **FUTURES K线数据**：用于获取交割合约的 1m 历史 K 线。
- **SWAP K线数据**：用于获取永续合约的 1m 历史 K 线（用于交割-永续价差计算）。
- **SPOT K线数据**（可选）：用于获取对应现货的 1m K 线（做基差收敛与三向套利参考）。

### 2. 合约生命周期解析与选择逻辑
交割合约由系统动态生成，具有明确的生命周期。管线必须具备对以下参数的无前视解析：
- **合约选择**：主要针对流动性最佳的**当季合约（Current Quarter）**与**次季合约（Next Quarter）**。每周、双周合约由于生命周期过短，摩擦成本太高，不作为主要回测目标。
- **合约命名规则**：解析 OKX 交割合约的代码规则（`{Base}-{Quote}-{YYMMDD}`，如 `BTC-USDT-260925`），并自动映射到其到期日 `2026-09-25 08:00 UTC`。
- **合约对齐序列**：根据时间轴自动判定当前时间点 `t` 对应的“当季合约代码”与“次季合约代码”，并在换月点无缝衔接。

### 3. 换月规则（Rollover Schedule）
为避免交割前夕的流动性陷阱，必须在数据管线中硬编码如下换月逻辑：
- **强制换月阈值**：**在旧交割合约到期日前至少 3 天（72小时）强制换月**，不得持有至交割日。
- **换月触发时间**：回测中一旦满足强制换月时间，逻辑自动平仓所有旧合约仓位，并在下一个 15m 开盘开仓新合约。

### 4. 序列拼接与指标计算方法
- **严禁使用直接拼接价格**：禁止直接将 `Future1_Price` 与 `Future2_Price` 拼接成一条伪价格序列并计算其绝对价格指标（如均线、通道、RSI），这会在换月点产生虚假的“跳变突破信号”。
- **价差序列法（Spread-First）**：回测管线必须优先计算价差时间序列：
  $$\text{Spread}_t = \text{Price}_{\text{Future}, t} - \text{Price}_{\text{Perp}, t}$$
  所有的统计均值、标准差、滚动分位数以及交易信号触发，**必须仅建立在价差序列 $\text{Spread}_t$ 自身上**。

## 三、交易成本与执行风险硬核审计

策略研究的成本配置必须严格遵守以下参数设定，任何低于此标准的模拟收益均为无效证据：

### 1. 双腿交易摩擦成本（四腿往返）
跨期套利/价差交易每次完整的建仓与平仓涉及两个合约的并发操作，共 4 腿交易：
- **建仓**：永续 Buy (Taker) + 交割 Sell (Taker)
- **平仓**：永续 Sell (Taker) + 交割 Buy (Taker)
- **硬性成本底线**：往返 4 腿合计费率与滑点损耗配置**必须设定为至少 0.32%**（以单腿 Taker 0.05% + 滑点 0.03% 为基准）。

### 2. 换月交易摩擦成本（Rollover Cost）
在持仓跨期交易换月时，需进行平旧开新操作（2 腿交易）：
- **换月摩擦**：旧交割平仓 (1 腿 Taker) + 新交割开仓 (1 腿 Taker)。
- **成本损耗**：单次换月额外扣除 **至少 0.16%** 的摩擦成本。

### 3. 关键执行风险列示
- **单腿成交风险（Leg Risk）**：由于 OKX 免费 API 不支持原子的组合单执行，策略以两条独立订单发出时，存在单边成交被动暴露方向风险的问题。
- **流动性枯竭与宽价差**：到期前流动性萎缩，盘口买卖价差变宽，Taker 滑点可能成倍放大。
- **交割手续费风险**：如果在到期时仍未完成换月，强制交割将触发高达 **0.05%** 的官方交割费率（相比于普通交易的 Maker/Taker 费率极高），此风险必须在回测中按违规扣除惩罚。

## 四、下载器与数据管线验收清单（Pre-Implementation Checklist）

在允许编写任何价差策略或修改下载器代码前，数据管线必须完全通过以下验收标准：

- [ ] **1. 下载器归档检查**：证明已在不下载付费数据的前提下，获取了至少 365 天、覆盖 20+ 个山寨币的 FUTURES、SWAP 1m 历史 K 线，且字段无缺失（覆盖率 $\ge 95\%$）。
- [ ] **2. 无前视换月审计**：证明在每个历史 bar `t`，策略只能知道当时已上市的交割合约代码及交易数据，不能提前读取未来上线的合约代码。
- [ ] **3. 强制换月测试**：回测系统必须能够展示，在每一个交割日前 3 天，所有旧合约的模拟仓位已被 100% 自动清空，没有任何交割费损耗。
- [ ] **4. 4 腿成本硬编码**：验证回测引擎中所有 Calendar Spread 策略的往返单次执行成本硬编码为 `0.0032`（即 0.32%）。
