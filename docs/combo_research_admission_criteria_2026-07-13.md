# 组合层研究准入门槛与防线规则 (G17)

**文档状态**：仅新建  
**文档路径**：`docs/combo_research_admission_criteria_2026-07-13.md`  
**基准约束**：  
- `approved_for_paper = []`  
- `safe_to_enable_trading = false`  
- **纪律红线**：严禁放宽硬性门槛以使回测曲线强行通过。凡是违反任何一项硬性准入规则的组合研究，一票否决，不得更新注册表。

---

## 一、概述

组合策略（Combo Strategy）的研究旨在将多个通过前置预检的弱期望特征、环境标签与过滤器融合为统一的头寸管理模型。为了防止由于“自由度倍增”带来的二次过拟合风险，本规范设立了严格的**组合层研究准入门槛**。

门槛划分为：
1. **硬规则 (Hard Gates)**：代码级强约束。不满足任何一项硬规则的组合研究直接判定为 **`invalid`**，严禁立项，严禁接入 `runner.py`。
2. **软性观察指标 (Soft Indicators)**：研究层分析指标。不作为硬性拦截，但作为最终研究卡评审的重要参考。

---

## 二、组合研究准入门槛明细表

| 评估指标 | 门槛数值/要求 | 判定类型 | 设立目的与物理限制依据 |
| :--- | :--- | :--- | :--- |
| **最少特征数量 (Min Features)** | $\ge 3$ 个 且 $\le 7$ 个 | **软指标** | 特征数太少（<3）无法体现多因子分散优势；特征数过多（>7）会导致参数爆炸和维度灾难，极易发生共线性过拟合。 |
| **单个特征最少事件数 (Min Events)** | 365天回测内 $\ge 30$ 次 | **硬规则** | 确保单个特征的信号触发频次具有统计显著性。事件数过少（如 $<30$ 次）极易受到少数随机大盈利交易的污染。 |
| **样本外 (OOS) 覆盖时长** | 至少 180 天 (或 $\ge 35\%$ 时序占比) | **硬规则** | 强制留出足够的纯样本外空间进行泛化验证。样本外时间过短无法检验策略的稳健性。 |
| **特征相关性上限 (Max Correlation)** | 任意两个特征相关系数 $r < 0.70$ | **硬规则** | 阻断共线性特征共振。如果 $r \ge 0.70$，说明两个特征在物理上是冗余的，组合时必须剔除其中一个。 |
| **单月收益贡献上限 (Max Single Month)** | 单月收益占比 $\le 25\%$ | **硬规则** | **必须包含“剔除 2024-11 牛市”的独立对照表现**。若单月贡献 $>25\%$ 且剔除后期望为负，强制否决，防止特化性拟合。 |
| **月度换手率上限 (Max Monthly Turnover)** | 每月总持仓换手率 $\le 60\%$ | **硬规则** | **物理限制依据**：对应平均持仓时间 $\ge 3$ 天。换手率过高（$>60\%$）会引发频繁开平仓，多腿 taker 摩擦会迅速吞噬全部净收益。 |
| **权重冻结规则 (Weight Freezing)** | IS 拟合结束后，OOS 验证前强制冻结 | **硬规则** | **防前视偏差依据**：权重分配一旦在 IS 数据上选定，在接入 OOS 阶段前必须完全锁定。禁止根据 OOS 结果反向微调权重。 |
| **风险禁止策略 (Risk Blocked Families)** | 组合中绝对禁止出现网格/马丁/加仓 | **硬规则** | 这类以尾部风险换取表面高胜率的逻辑在组合层同样会被放大，必须在特征池准入端直接拦截。 |

## 三、硬规则 (Hard Gates) 详细执行规范

在组合层研究卡申请中，必须对以下硬规则进行机械式审计：

### 1. 特征相关性审计 (Correlation Audit)
- **执行方式**：利用 `FeatureBar` 序列计算每个 Feature 之间的 Pearson 相关系数矩阵。
- **拦截阈值**：如果矩阵中存在任意元素（对角线除外）的值绝对值 $\ge 0.70$，则该组合方案不能通过 preflight 审核。研究员必须移除相关性最高的那一个 Feature。

### 2. 剔除 2024-11 集中度审计 (Concentration Audit)
- **执行方式**：统计组合在整个回测区间（如 365天）内的月度 PnL。
- **拦截阈值**：
  - 计算公式：
    $$\text{Concentration Ratio} = \frac{\text{PnL}_{2024-11}}{\sum \text{PnL}_{\text{all months}}}$$
  - 如果 `Concentration Ratio > 0.25`，则启动**扣除对照测试**。如果剔除 2024-11 的净收益后，策略总收益为负，或者夏普比率低于 $0.5$，直接否决，标记为 `rejected`。

### 3. 月度换手率与最低净期望均值审计 (Turnover & Expectancy Audit)
- **执行方式**：回测中统计每一笔成交，并计算交易摩擦。
- **拦截阈值**：
  - 月度换手率公式：
    $$\text{Turnover}_{\text{monthly}} = \frac{\sum \text{Executed Notional}_{\text{month}}}{\text{Average Equity}_{\text{month}}}$$
  - 若 `Turnover > 0.60`，或者单笔平均净收益（Net Expectancy per Trade）低于 $0.16\%$，判定为 **“成本吞噬型失败”**，直接一票否决。

### 4. 权重确定与 OOS 阻断审计 (Weight Freezing Audit)
- **执行方式**：采用“时间轴单向切分”或“前向滚动（Walk-Forward）”拟合。
- **拦截阈值**：任何试图将 OOS 的回测绩效（如 OOS 夏普比率）作为参数回传给权重优化器（Optimizer）的代码结构，在单元测试 `test_combo_research_safety.py` 中必须被断言阻断。

## 四、软性观察指标 (Soft Indicators) 引导意义

软性观察指标不作为拦截研究的硬性红线，但对于评估策略的实盘可用度至关重要：

1. **组合夏普比率 (Sharpe Ratio)**：  
   - OOS 表现期望：$\text{Sharpe} \ge 1.50$。若低于此数值，说明多特征组合未实现良好的风险分散。
2. **最大回撤控制 (Max Drawdown)**：  
   - OOS 表现期望：$\text{MaxDD} \le 15\%$（在使用规定的 $1\times$ 基础风险杠杆下）。
3. **单特征与组合整体胜率 (Win Rate)**：  
   - 期望组合胜率 $\ge 50\%$。对于趋势跟随特征，允许单特征胜率在 $35\% - 45\%$ 之间，但组合后通过投票机制，期望总体胜率应能提升至 $50\%$ 以上。
4. **币对相关度分布 (Pairwise Correlation)**：  
   - 观测组合在不同山寨币板块（如 Layer1, DeFi, Meme）的暴露分布。如果头寸过度集中在单一板块，虽然能通过硬性相关性过滤，但存在隐性板块共振风险，需要在研究报告中做出警示。
