# 10U Event Trend v2 扫止损与盈利回撤指标审计

## 审计结论

旧口径低估了扫止损，并把资金费率混入盈利行情捕获率。修正发生在前瞻账本仍为 0 条信号时；入场、止损价格、仓位、退出和风险参数均未改变。

## 扫止损口径

正式指标只以 `hard_disaster_stop` 为分母。结构收盘失效和 4H 追踪退出是策略主动退出，不与硬止损混为一类。

一笔硬止损被认定为“扫损后恢复”需要：

1. 从止损后的下一根 1H K 线开始观察，避免同一根 OHLC 无法确定高低点先后顺序；
2. 在该交易原定 48 小时持仓窗口结束前；
3. 价格重新到达原始入场价。

另行记录更严格的“恢复到入场价后再走出 1R”，但它不代替回到入场价的基础扫损指标。

## 已污染历史窗口的诊断变化

历史封闭筛选仍只有 3 笔实际交易，不能证明策略有效。其中唯一一笔硬止损为 LAB：

- 入场：`2026-05-07T07:00:00Z`
- 硬止损：`2026-05-07T11:00:00Z`
- 止损后重新到达入场价：14 小时后
- 止损后达到入场价再盈利 1R：否

旧指标只观察止损后 12 小时，因此错误报告为 0% 恢复；正确口径为 1/1，即 100%。样本只有一笔，不能估计长期扫损率，但足以证明旧口径存在假阴性。

## 盈利捕获率口径

正式捕获率为：

```text
退出时已实现的毛价格方向收益 / 持仓期最大有利价格方向收益
```

资金费率、手续费和滑点分别报告，不允许资金费率收入抬高“抓住行情”的分数。修正后的两笔历史盈利交易价格捕获率约为：

- LAB：70.19%
- RAVE：91.70%
- 中位数：80.94%

旧的含净收益口径中位数为 81.75%。

## 权益峰值与回撤

逐小时权益峰值现在按“该时点立即平仓后的可执行权益”计算，同时预扣估计出场滑点和手续费。历史诊断的峰值由约 245.81U 调整为约 245.62U；最终权益不变，仍约 205.01U。此修正只影响峰值、利润保留率和回撤测量，不改变成交。

## 决策

v2 继续保持冻结前瞻观察，不根据这唯一一笔历史扫损修改止损。正式样本若硬止损后回到入场价的比例超过 35%，v2 将验收失败；失败后不能通过调宽止损或修改恢复窗口挽救。
