## 候选 A：日内动量反转 (Intraday Momentum Reversal)

### 利润来源假设

极端日内收益（单根大 K 线）倾向于在下一根 K 线反转。这是传统市场中 well-documented 的 anomaly。

### 有效行情

- 高波动市场中的过度反应
- 不适合：趋势行情（极端收益可能继续）

### 已有策略未覆盖的原因

当前策略都是趋势跟随或突破策略，没有专门做反转的。range_revert 是区间反转，不是日内动量反转。

### 需要的数据

- 15m K 线的单根收益率（已有）

### 无未来泄漏回测可行性

✅ 可以用当前数据完成。只看前一根 K 线的收益。

### 可能的失效机制

- 趋势行情中极端收益继续（不反转）
- 交易成本侵蚀小收益
- 信号太少

### 实现方式

```python
# 检测极端日内收益
bar_return = (close - prev_close) / prev_close

# 极端做空信号：单根涨幅 > 3%，下一根做空
if bar_return > 0.03:
    signal = short_signal  # 反转

# 极端做多信号：单根跌幅 > 3%，下一根做多
if bar_return < -0.03:
    signal = long_signal  # 反转
```

## 候选 B：成交量-价格背离 (Volume-Price Divergence)

### 利润来源假设

当价格创新高但成交量下降时，表明买盘力量减弱，价格可能反转。

### 有效行情

- 趋势末期的力竭
- 不适合：趋势初期（量价齐升是健康的）

### 已有策略未覆盖的原因

当前策略没有专门检测量价背离。volume_ratio 只看绝对量，不看量的变化趋势。

### 需要的数据

- 15m K 线的成交量和价格（已有）

### 无未来泄漏回测可行性

✅ 可以用当前数据完成。量价背离只使用历史数据。

### 可能的失效机制

- 趋势行情中量价背离后价格继续上涨
- 信号定义模糊，容易过拟合
- 需要多个时间框架确认

### 实现方式

```python
# 计算价格和成交量的趋势
price_trend = close / close_20_bars_ago - 1
volume_trend = volume_sma_5 / volume_sma_20 - 1

# 量价背离：价格上涨但成交量下降
if price_trend > 0.02 and volume_trend < -0.2:
    signal = short_signal  # 看跌背离

# 量价背离：价格下跌但成交量下降（卖盘衰竭）
if price_trend < -0.02 and volume_trend < -0.2:
    signal = long_signal  # 看涨背离
```

## 候选 C：波动率状态切换 (Volatility Regime Switching)

### 利润来源假设

市场在低波动和高波动状态之间切换。当从低波动切换到高波动时，趋势策略有效；当从高波动切换到低波动时，区间策略有效。

### 有效行情

- 波动率状态转换期
- 不适合：稳定在某一波动率状态

### 已有策略未覆盖的原因

当前策略没有根据波动率状态动态切换策略类型。transition regime 是趋势强度，不是波动率状态。

### 需要的数据

- 15m K 线的 ATR（已有）

### 无未来泄漏回测可行性

✅ 可以用当前数据完成。波动率状态只使用历史数据。

### 可能的失效机制

- 波动率状态判断滞后
- 状态切换频繁导致交易成本高
- 需要精确的状态定义

### 实现方式

```python
# 定义波动率状态
atr_ratio = atr_5 / atr_50
if atr_ratio < 0.7:
    volatility_state = "low"
elif atr_ratio > 1.3:
    volatility_state = "high"
else:
    volatility_state = "normal"

# 根据状态选择策略
if volatility_state == "high":
    # 使用趋势策略
    signal = trend_signal
elif volatility_state == "low":
    # 使用区间策略
    signal = range_signal
```
