# 年度策略研究数据准入合同

更新时间：2026-07-11

这不是策略清单，而是任何新数据在进入年度研究前必须满足的验收合同。目的
是杜绝“有几天数据、做出一张回测表”被误当作可执行证据。

## 硬门槛

1. 至少连续 365 天，优先 24 个月；时间戳必须为 UTC，不能插值伪造缺口。
2. 用于 OKX 执行收益结论的数据，必须来自 OKX 同一永续合约；外部交易所数据
   只能标为 `research_proxy_only_not_okx_execution`。
3. 每个字段覆盖率至少 95%；低覆盖字段（例如仅最近几天的大户多空比）不能用于
   年度策略特征。
4. 必须保留原始文件、来源、合约代码、频率、首末时间戳、下载日期与字段说明。
5. 数据必须可由任何研究者通过公开、免费的官方端点或公开归档重复获取；不得依赖付费 API、订阅数据集、购买历史数据或人工导出数据。
6. 数据接入后先运行：

```bash
python research_data_gate.py --out reports/research_data_gate.json
```

只有 `annual_execution_eligible=true` 的数据才可进入 365 天 OKX 执行策略验证。

## 按优先级补齐的数据

| 优先级 | 数据 | 最低频率 | 目标覆盖 | 能回答的问题 |
|---|---|---:|---:|---|
| P0 | OKX 永续 OI 历史 | 5m 或 15m | 24个月 | 杠杆是否在价格走势前后异常累积或出清。 |
| P0 | OKX 资金费率历史 | 每次结算 | 24个月 | 拥挤持仓是否带来可交易的后续反转/延续。 |
| P1 | OKX 现货与永续的可对齐价格 | 1m/5m/15m | 24个月 | 基差是否足以覆盖成本，并在同一市场可执行。 |

可回放的长期清算事件、逐笔订单流、长期盘口与链上数据常需要付费历史数据；在当前项目约束下它们**不属于待补数据源，也不得作为新策略前提**。

## 接入顺序

只允许使用已达标的免费公开数据提出有明确经济机制的候选；不允许为了获得某项
高频历史数据而转向付费来源，也不允许在数据不达标时通过调阈值、挑币种或扩大杠杆
继续试验。策略必须用形成期/样本外分离、真实手续费滑点和跨窗口验证，才有资格进入模拟盘。
