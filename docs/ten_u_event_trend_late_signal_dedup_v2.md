# v2 迟到信号一次性拒绝审计

若观察器停机或数据源延迟，某个信号可能在其计划入场时间之后才被发现。该信号必须拒绝，不能事后
补入可交易前瞻账本。但拒绝本身也必须可追踪且只能计数一次。

修正前，迟到信号不进入主账本，因此后续每次全量重算都会再次把它识别为“从未见过”，造成重复拒绝
和重复计数。修正后，每个迟到信号的 `{symbol, entry_ts}` 安全键写入刷新审计哈希链：

- 第一次迟到发现：记录并计入 `late_signal_records_rejected`；
- 后续再次扫描：识别为 `previously_rejected_late_signal_records`，不重复计数；
- 安全键不包含退出、收益、MFE、MAE 或净值；
- 同一个迟到键不能在审计链中出现两次；
- 它永远不能进入可交易信号账本或成熟度样本。

修正时前瞻信号和迟到信号均为 0，因此没有删除、重分类或补写任何既有样本。
