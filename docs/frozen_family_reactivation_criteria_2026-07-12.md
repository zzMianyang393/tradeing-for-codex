# 冻结与风险受阻策略家族长期重启条件

**审查日期**：2026-07-12  
**仅新建**：`docs/frozen_family_reactivation_criteria_2026-07-12.md`  
**核心宗旨**：规定当前处于 `frozen` 或 `risk_blocked` 状态的五大策略家族为什么被封禁，以及在何种严苛的先决条件下才可以被重新考虑，防止研究资源在无意义方向上发生漂移。

---

## 一、网格 / 马丁 / 锁仓 / 亏损加仓家族

### 1. 当前为什么冻结/受阻？
- **状态**：**`risk_blocked`（风险受阻）**
- **冻结原因**：这类策略的本质是“用尾部爆仓风险换取日常高净胜率”。在震荡行情中，网格/马丁通过逆势加仓拉低均价，表现出极其平滑的上升权益曲线；但一旦遭遇单边趋势（如大牛市、大熊市或断头跌），仓位会呈几何级数累积，最终导致账户彻底爆仓（Margin Call / Liquidation）。这违反了量化交易防范单次黑天鹅的底线风险管理规则。

### 2. 重启先决条件
如果未来要重新讨论此类策略，必须**同时满足**以下条件：
- **最大仓位硬上限**：策略必须在回测和实盘中被硬性限制总最大敞口（例如，无论行情如何波动，累积加仓次数上限为 3 次，总名义本金不得超过单次基础仓位的 3 倍）。
- **极端的清盘压力测试**：回测系统必须引入历史上最极端的单边行情（如 312 暴跌、FTX 破产、LUNA 归零等）作为强制历史测试段，策略在此区间内必须触发全局止损而退出，且清盘后的总账户最大回撤不得超过 **15%**。
- **严禁使用亏损加仓作为阿尔法**：必须先证明开仓信号本身具备正期望值，网格/加仓只能作为执行层的优化工具，不得用“亏损加仓拉低均价”来强行掩盖信号本身的无能。

## 二、机器学习 / 动态路由家族

### 1. 当前为什么冻结？
- **状态**：**`frozen`（冻结）**
- **冻结原因**：在没有被证明具备独立正期望（Alpha）的单体信号前，使用复杂的机器学习（LSTM、XGBoost、强化学习）或多信号动态路由，实际上只是在对历史噪声进行极度复杂的参数拟合。最终高收益的机器学习模型在进入样本外（OOS）后会发生剧烈的过拟合崩溃。

### 2. 重启先决条件
如果未来要重新讨论机器学习或多信号路由，必须**同时满足**以下条件：
- **单体信号先决**：**必须至少有 2 个通过了 365天 OOS 验证、具有独立正期望的规则型单体策略**作为 ML 模型的输入特征。ML 或动态路由只能扮演“过滤器”或“组合分配器”的角色，严禁让模型直接去价格噪声中凭空学习开仓信号。
- **严格的时间滑窗交叉验证（Walk-Forward Validation）**：模型必须进行滚动训练与验证（例如：训练 6 个月，样本外滚动测试 2 个月），且必须展示在 OOS 区间内，ML 路由的期望收益显著高于均权组合（Equal Weight），否则直接降回均权。
- **严禁依靠参数搜索伪造阿尔法**：模型的超参数（如树深度、学习率、特征数）必须在预注册中锁定，禁止通过跑大量随机种子或参数网格来“挑选”表现最好的种子。

## 三、外部事件 / 新闻舆情 / 宏观家族

### 1. 当前为什么冻结？
- **状态**：**`frozen`（冻结）**
- **冻结原因**：目前缺乏公开、免费且能连续覆盖 365天 以上的结构化历史事件数据库。此外，新闻发布时间戳在回测中极难与 K 线进行无前视偏差的对准，事件发生时的剧烈滑点也无法在回测中还原。

### 2. 重启先决条件
如果未来要重新讨论事件驱动交易，必须**同时满足**以下条件：
- **免费公开数据门槛**：必须提供一份由官方或可靠公开源提供的、连续 365天 以上的结构化历史事件文件，且其发布时间戳（毫秒级，UTC）可被任何人免费复现下载。
- **硬性延迟入场与惩罚性滑点**：回测必须假设在事件时间戳之后**至少延迟 1 分钟**才允许以当时的市价（Taker）入场，且单腿滑点扣除必须设为平时均值的 **3–5 倍**，以模拟事件发生时流动性瞬间枯竭的真实执行状况。
- **严禁建议付费数据**：不得建议购买商业舆情 API 或使用需要订阅的历史数据库来绕过此限制。

## 四、Tick 级 / 盘口高频家族

### 1. 当前为什么冻结？
- **状态**：**`frozen`（冻结）**
- **冻结原因**：OKX 官方免费 API 仅支持极短的实时盘口快照（Book）和 7 天滚动成交（Trades），没有提供免费的长期历史 Tick 归档。此外，高频套利和盘口剥头皮对低延迟（网络延迟、API 限频）极度敏感，在常规回测系统（如 15m/1m）中根本无法真实复现执行质量。

### 2. 重启先决条件
如果未来要重新讨论高频或盘口策略，必须**同时满足**以下条件：
- **完整订单流归档**：必须获得 OKX 官方提供的、至少 365天 的免费 Tick/Orderbook 历史数据库。
- **实盘执行延迟模拟**：回测系统必须引入至少 **100ms - 200ms** 的随机网络与处理延迟，且市价单必须在盘口深度中真实消耗挂单（Market Impact），不能假设以当时的 Mid-price 成交。
- **滑点与排队模拟**：限价单必须模拟在 OKX 队列中的排队位置（Queue Position），不能假设挂单即成交。

## 五、期权 / 波动率家族

### 1. 当前为什么冻结？
- **状态**：**`frozen`（冻结）**
- **冻结原因**：OKX 免费接口提供的期权链（Option Chain）、行权量及隐含波动率（IV）历史数据覆盖不足 30 天，无长期月度归档。

### 2. 重启先决条件
如果未来要重新讨论期权波动率对冲，必须**同时满足**以下条件：
- **期权链数据达标**：必须提供至少 365 天、包含不同到期日和行权价的 OKX 期权 K 线与 IV 历史数据库，字段覆盖率 $\ge 95\%$。
- **期权希腊字母（Greeks）与保证金压力测试**：期权策略回测必须完整计算 Delta, Gamma, Vega, Theta，且必须在回测中硬性加入 OKX 官方的**组合保证金账户（Portfolio Margin）**规则，模拟在极端波动下保证金不足被强制减仓的清盘线。
