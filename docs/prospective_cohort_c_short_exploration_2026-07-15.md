# Cohort C: Volatility-Expansion Short Exploration

`daily_volatility_expansion_short_exploration_v1` is a post-hoc research
hypothesis, not a historically approved feature. The historical stability
audit found OOS direction asymmetry in the frozen two-sided volatility rule;
that observation cannot justify retrospective short-only backtesting or paper
approval.

The exploration is activated only from `2026-07-16 00:00 UTC`, strictly after
the available data cutoff. It emits a signal only when the frozen daily
volatility-expansion conditions occur with a bottom-quartile close in completed
`高波动转换`, and it records only metadata. It never records price, return,
position, order, exit, or outcome fields.

Cohort C is separate from Cohort B. It cannot amend Cohort B, add a paper
candidate, or enter a combination. Any evaluation is deferred until the
separate prospective observation period matures under a future sealed protocol.

## Observation Publication

The normal daily cycle is dry-run. After the activation boundary, a formal
Cohort C observation can be published only with both explicit flags:

```powershell
py -3 prospective_daily_observation_cycle.py --commit-data --commit-cohort-c-observations --fetch-workers 4
```

The second flag is rejected unless `--commit-data` is also present. Publication
still requires a valid, gap-free, duplicate-free data-quality report and a
strictly append-only new signal. It only records observation metadata and then
generates the sealed-evaluation preflight; it does not create a paper or live
trading permission.

## Canonical Artifacts

The only canonical staging ledger is
`reports/staging_cohort_c/prospective_cohort_c_short_exploration_ledger.json`.
When a new observation is validly published, the formal artifacts are the
root-level Cohort C ledger, registry, maturity audit, and sealed-evaluation
preflight generated by `prospective_cohort_c_refresh_pipeline.py`.

The legacy root-level file named
`reports/prospective_cohort_c_short_exploration_staging_ledger.json` is not an
input to the dashboard or publication pipeline and must not be used to infer
the current Cohort C state. This prevents a stale manual report from competing
with the append-only pipeline staging ledger.
