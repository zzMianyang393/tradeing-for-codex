## 四、下一阶段正确任务

### 任务 1：先由 Claude Code 提出 3 个新策略候选

候选必须是逻辑上独立的新策略，不得只是给旧策略增加 RSI、成交量、均线阈值。

每个候选必须说明：

1. 利润来源假设是什么。
2. 在什么行情中应该有效。
3. 为什么已有策略没有覆盖它。
4. 需要哪些数据。
5. 是否可以用当前 15m 数据完成无未来泄漏回测。
6. 可能的失效机制是什么。

优先考虑以下策略族：

- 跨币种相对强弱/弱势轮动，而不是单币绝对指标。
- 多周期确认策略，例如 4h 状态 + 15m 入场，但必须处理 BTC/ETH 数据覆盖差异。
- 事件后的波动衰减或反转策略，要求明确事件定义和净 PnL 标签。
- 资金费率策略只能作为 90 天研究候选，不能作为 365 天结论，除非找到可信的历史数据源。

### 任务 2：实现无参数搜索的策略基线

每个候选只实现一套保守默认规则，输出：

- 90 天、180 天、365 天收益。
- 最大回撤、胜率、盈亏比、交易次数。
- 12 个月拆分。
- `by_symbol`、`by_reason`、`by_regime`。
- 固定币种集合和全部币种结果。
- 手续费、滑点后的净 PnL。

禁止先做参数网格搜索。

### 任务 3：固定币种宇宙

至少测试以下宇宙：

1. BTC/ETH/SOL/BNB/XRP 主流集合。
2. 固定 5 个高流动性山寨币集合。
3. 全部可用 15m 币种。

如果策略只在一个宇宙盈利，标记为“币种集合敏感”，不得进入优化。

### 任务 4：硬性淘汰标准

任何候选满足以下任一条件，直接淘汰：

- 365 天亏损。
- 180 天和 90 天同时亏损。
- 全部币种与主流币集合方向相反。
- 单个月份或少数交易贡献大部分利润。
- 最大回撤超过 45%。
- 交易次数太少，无法形成统计证据。
- 需要针对某个月份、某几个币、某个窗口单独改参数。

## 五、只有通过基线后才允许做的事

只有新策略通过无参数基线，才允许：

1. 做小范围参数敏感性测试，参数最多 3 个。
2. 做滚动 walk-forward。
3. 做 12 个月稳定性分析。
4. 与其他已通过策略组合。
5. 进入模拟盘观察。

模拟盘不是为了证明一个明显亏损策略会赚钱，而是验证执行、滑点、订单状态、对账和暂停机制。

## 九、全面失败分析（2026-07-11 更新）

### 13 个候选的失败分类

| 失败类型 | 涉及候选 | 核心原因 |
|---|---|---|
| 全窗口亏损 | 相对强弱、多周期确认、波动率状态、低换手趋势、相对强弱持续 | 假设本身在当前数据中不成立 |
| 成本敏感 | 量价背离、日内反转 | 高频交易被手续费+滑点吃掉 |
| 跨币种矛盾 | 主动买卖量吸收 | BTC亏/ETH赚，方向不一致 |
| 样本不足 | 多日急跌反转、资金拥挤反转、波动率压缩突破、量能衰竭反转 | 事件定义太严格或条件内在矛盾 |
| 币种集合敏感 | 震荡反转做空 | 只在特定币种集合盈利 |
| 标签/路由污染 | CP 空头、多周期确认（旧版） | 验证器问题（已修复） |

### 根本问题诊断

1. **数据覆盖不足**：山寨币只有24个月数据（2024-07至今），BTC/ETH有30个月。365天窗口消耗了大部分数据，样本外验证空间极小。

2. **成本结构严苛**：OKX taker fee 0.05% + 滑点 0.03% = 单边 0.08%，双边 0.16%。任何持有时间短于数天的策略都需要至少 0.32% 的平均盈利才能覆盖成本。

3. **市场结构假设未验证**：所有候选都假设"某种技术形态会重复出现并带来正期望"，但没有证据表明这些形态在加密市场中具有持续的预测能力。

4. **信号 vs 噪声**：在15分钟K线上，技术指标的信号/噪声比极低。日度聚合可能改善，但会减少交易次数至统计不显著的水平。
