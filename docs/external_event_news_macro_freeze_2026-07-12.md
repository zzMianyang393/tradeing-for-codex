# 外部事件、新闻舆情及宏观方向冻结说明

**审查日期**：2026-07-12  
**受阻方向**：宏观经济数据事件驱动、新闻舆情爬虫交易、币种利好/事件利空自动交易、项目财报/代币减半等外部事件驱动交易。  
**仅新建**：`docs/external_event_news_macro_freeze_2026-07-12.md`

---

## 一、为什么当前这些方向不适合 OKX EA 研究？

虽然外部宏观事件（如美联储利率决议、CPI公布）和行业/币种新闻（如项目方利好、交易所上币公告）对加密货币价格有显著的短期冲击，但在本项目当前的研究框架和技术约束下，这些方向**目前无法产生可靠、可验证、无偏差的量化执行结论**。

核心原因如下：

### 1. 免费且可复现的历史数据库匮乏
- **准入门槛违背**：根据《年度策略研究数据准入合同》，所有用于回测的数据必须可由任何研究者通过公开、免费且官方的端点重复下载，且覆盖至少 365 天。
- **现状**：目前市面上没有免费、公开且包含结构化历史新闻/宏观事件（精确到毫秒级发布时间、事件权重、利好/利空分类）的权威数据库。使用人工收集或不完整的零散数据做出的回测，无法满足 100% 公开可复现的标准。

### 2. 发布时间戳对齐与执行时效性极度困难
- **时序对准难题**：在回测中，外部新闻的“发布时间”通常是新闻网站上的编辑时间戳，这与该新闻被爬虫抓取、被算法解析并最终转化为下单指令的**实际延迟时间**存在巨大差异。
- **滑点与滑移风险**：事件驱动交易极度依赖“时效性”。在真实市场中，重大事件发生后的第一秒内，市场价格通常会瞬间跳空（Gap），流动性瞬间枯竭。回测若假定能以事件发生时刻的 K 线收盘价或下一根 15m 开盘价成交，将产生严重的乐观滑点偏差。

### 3. 极高风险的前视偏差（Look-Ahead Bias）
- **现象**：许多事件驱动策略在回测中表现优异，是因为回测引擎在 `t` 时刻使用了新闻库中带有 `t` 日期标记的数据。
- **实质**：许多利好或宏观事件虽然发生在 `t` 日，但公众实际获知时间、或数据源更新时间可能在几小时后。在回测中无法完美还原“在下单那一刻该新闻是否已 100% 可见”，极易引入隐蔽的前视偏差，导致“马后炮”式的高收益。

### 4. 缺乏 OKX 市场同源性
- **外部数据污染**：宏观数据与新闻并非 OKX 交易所的原生微观结构数据（如 Funding、OI、K线）。
- **执行结论冲突**：在外部事件发生时，不同交易所的微观反应不同。无法仅凭外部宏观数据作为 OKX 单一平台执行收益的直接证据。

## 二、后续重启该研究的前置准入门槛

本项目**并不禁止**在未来条件成熟时研究上述方向，但任何关于外部事件/新闻舆情方向的重新评估，**必须先通过以下数据门槛审计**：

1. **结构化历史归档**：必须提供一份由免费、公开源提供的，至少连续 365 天、包含精确发布毫秒时间戳（UTC）、且无插值伪造的结构化新闻/宏观数据库。
2. **延迟与滑点压力测试**：回测系统必须引入“事件发布后至少延迟 N 秒且伴随惩罚性滑点（如 0.1%–0.5%）”的滑移入场语义，证明扣除极端滑点后仍有正期望。
3. **不得推荐付费替代方案**：严禁通过购买商业舆情 API、订阅 Bloomberg/Reuters 等付费历史终端数据来满足回测，这违反免费公开可复现的硬规则。

## 三、最终状态判定

- **处理状态**：**`frozen` / `data_blocked`**
- **执行约束**：
  - 严禁在当前项目中提出或编写任何基于舆情爬虫、宏观日历或新闻事件的交易策略代码。
  - 不得在 `runner.py` 或回测模块中预留相关接口。
  - 任何此类方向的提议在满足第二部分的前置门槛前，直接予以驳回。
