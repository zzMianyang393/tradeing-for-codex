# Open Interest (OI) 去杠杆冲击数据可行性审查报告

**审查日期**：2026-07-12  
**仅新建**：`docs/oi_deleveraging_data_feasibility_2026-07-12.md`  
**审查目的**：判定持仓量（OI）周期与去杠杆冲击策略，是否具备符合项目数据准入合同的 365天 以上免费历史数据，并明确划定研究边界。

## 一、各数据维度覆盖与合格性审计

根据 `reports/research_data_gate_post_download.json` 及 OKX 官方免费 API 规范，对该方向所需的数据维度进行逐项资格审计：

| 数据维度 | 覆盖天数 | 是否满足 365天 | 数据状态 | 审计决策与原因 |
| :--- | :--- | :--- | :--- | :--- |
| **日线持仓量 (Daily OI)** | 555 天 (24币种) | ✅ **是** | **合格 (Eligible)** | 通过官方 `rubik` 日频历史接口获取，无插值，覆盖率符合要求，可用于收益审计。 |
| **15m 高频持仓量** | 约 30 – 90 天 | ❌ **否** | **数据受阻 (data_blocked)** | OKX 免费 API 对高频 OI 分页限制严重，无长期归档。**禁止用于 365天 收益回测**。 |
| **成交量 (K线 Volume)** | 730 天 (28币种) | ✅ **是** | **合格 (Eligible)** | 直接提取自 15m/1d OHLCV，覆盖完整。 |
| **历史市值 (Market Cap)** | 0 天 (无) | ❌ **否** | **数据受阻 (data_blocked)** | OKX 官方不提供历史每日市值归档，在不引入付费源的前提下，无法作为因子。 |

## 二、基于“日线 OI + OHLCV”的低频版本可行性

虽然高频 OI 和清算数据受阻，但利用**已达标的“日线 OI + 15m/1d OHLCV”**，理论上可以构建一个免除前视偏差的低频去杠杆过滤器：

### 1. 因子构建
- **持仓变化率（Daily OI Change）**：
  $$\Delta \text{OI}_t = \frac{\text{OI}_t - \text{OI}_{t-1}}{\text{OI}_{t-1}}$$
- **持仓量/成交量比值（OI-to-Volume Ratio）**：
  $$\text{OIVR}_t = \frac{\text{OI}_t (\text{USD})}{\text{Daily Volume}_t (\text{USD})}$$
  该比值可作为市场相对杠杆密度的代理指标（成交量取自 K 线，日线 OI 取自官方数据）。

### 2. 时序对齐与入场语义（防范前视）
- **数据可用点**：日线 OI 数据虽然时间戳记为当天，但根据 OKX 结算惯例，实际数据可用时间为 **当天 16:00 UTC**。
- **入场时间限制**：信号计算只能在 16:00 UTC 之后触发，回测中**最早入场时间必须设定为当天 16:15 UTC** 的 15m K 线开盘价，严禁在 16:00 之前偷看信号。

### 3. 禁止高频/Tick 级审计
- 明确禁止任何涉及 Tick 级订单流、逐笔爆仓单、实时盘口不平衡的高频套利收益审计。此类策略在缺乏免费 365天 Tick 归档和真实网络延迟模拟的前提下，回测结论 100% 为假显著。

---

## 三、审查判定结论

针对不同的研究子方向，给出的最终判定如下：

### 1. 15m 高频 OI 及清算驱动策略
- **判定**：**数据受阻 (data_blocked)**
- **理由**：历史数据严重不足 365 天，且无免费公开归档。

### 2. HFT / 盘口订单流去杠杆策略
- **判定**：**数据受阻 (data_blocked)**
- **理由**：Tick/Orderbook 覆盖为零，延迟无法复现。

### 3. 基于“日线 OI + OHLCV”的低频去杠杆过滤器（R3 行情状态过滤器）
- **判定**：**可研究（Eligible for Research，限 meta-only）**
- **要求**：
  1. **仅限作为不交易过滤器（No-Trade Filter）**：例如当 $\text{OIVR}_t$ 处于历史 95% 极值且日内波幅变宽时，强制关闭其他趋势策略的开仓，不产生主动交易信号。
  2. 回测摩擦成本配置单边 Taker 费加滑点必须 $\ge 0.08\%$。
  3. **严禁批准为 paper-eligible 策略**。
