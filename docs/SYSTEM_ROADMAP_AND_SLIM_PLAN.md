# 全系统路线图、进度与瘦身计划

日期：2026-07-17（约束修订同日）  
目标定义：**可研究策略 → 回测验证 → 本地 paper →（后期）交易所模拟盘见效 →（最后）小资金实盘**  
（不是「纯研究器」）

---

## 0A. 操作者硬约束（决策层必须遵守，2026-07-17）

以下约束优先于旧阶段表中「提前接 demo / 多币挑标的」表述。

### 晋级顺序与运行环境

| 阶段 | 在哪跑 | 允许 | 禁止 |
|------|--------|------|------|
| 现在～中期 | **本机 + 可同步到服务器的本地 paper** | 回测、本地 paper、部署工件 | 本机交易所下单、本机配交易密钥 |
| 后期 | **仅服务器** | OKX **模拟盘**（agent 配密钥；见效后才谈实盘） | 本机跑模拟盘；未见效就开实盘 |
| 最后 | **仅服务器** | 小资金实盘（agent 配密钥） | 本机实盘；跳过模拟盘 |

- **模拟盘与实盘都放最后，且只在服务器由 agent 配置 API Key/密钥。**  
- **本机决策范围**：研究质量、本地 paper 稳定性、可部署代码与 handoff 文档；**不**做本机 demo smoke / 密钥联调。

### 标的宇宙（生产与可晋级研究）

- **默认：只做 BTC + ETH**（`BTC-USDT-SWAP` / `ETH-USDT-SWAP`）。  
- **或**固定全宇宙、禁止事后挑币；**禁止**从多币回测里 cherry-pick 赢家币再上盘。  
- 原因：OKX 模拟盘可交易合约少；依赖冷门/特殊合约（如 RAVE/LAB）的路径**不能**作为模拟盘→实盘主链。  
- 10U 旧路径若绑定 RAVE/LAB：仅可作**本地 paper / 信号实验**；**不得**作为模拟盘/实盘晋级标的，除非规则改写到 BTC/ETH。

### 资金标准（回测 = 实盘起步逻辑）

| 项 | 规则 |
|----|------|
| 默认起始权益 | **10 USDT** |
| 因资金过小（最小名义、张数、手续费地板）导致回测失真时 | 可升至 **100–500 USDT** 做**资本约束敏感性**对照 |
| 硬顶 | **最高 500 USDT**；禁止用更大本金刷「好看曲线」再声称可 10U 起步 |
| 报告要求 | 凡升高本金，必须同时报告 **10U 基线**（或写明 10U 不可交易原因：min notional 等） |

### 加权总进度（主观）

**约 35%–40% → 100%「可自用自动交易系统」。**

- 若只论「研究工厂」：约 **70%+**（工具齐，结论多为负）

- 若论「执行系统骨架」：约 **40%**  

**最大缺口不是再写 100 个 audit，而是：生产主链 + 可交易策略宇宙 + 交易所自动执行闭环。**

## 4. 后续方向（按阶段，服务「自动交易系统」）

### 阶段 0 — 瘦身与冻结边界（0.5–1 天）

- [ ] 外置/删除 `reports/candidate_pool`（~5GB）  
- [ ] 外置可选 `data/basis`、`calendar_spread*`  
- [ ] `git rm --cached` data/reports；push 瘦代码  
- [ ] 文档声明：日常入口只有 `prod/` + `runner.py`

**出口：** 仓库 < 百 MB 级可推；本地仍可研究。

### 阶段 1 — 本地主链 MVP（主线，不含交易所模拟/实盘）

目标：`信号 → 风控 → 本地账本开平 → 状态` 可定时跑；规则与 **BTC/ETH + 10U（≤500）** 对齐。

1. **数据刷新守护**  
   - 生产向：BTC/ETH（15m 或策略所需周期）+ funding  
   - 旧 10U RAVE/LAB 刷新仅作遗留实验，**不**进晋级链  
2. **本地 paper 主循环产品化**  
   - 定时 `paper-cycle`、锁、halt、日志、权益  
3. **执行模式（本阶段）**  
   - `local_paper`：**默认且唯一默认**  
   - `okx_demo` / `okx_live_capped`：**关闭**；不接策略主循环  
4. **回测纪律落地**  
   - 默认 `start_equity=10`；资本敏感性仅 100/500 两档对照  
   - 宇宙：BTC+ETH 或固定全集不挑币  

**出口：** 无人值守本地 paper 稳定运行；至少一条 **BTC/ETH** 规则在 10U（及必要时 ≤500 对照）下有完整回测+本地 paper 指纹。  
**已落地代码：** `prod/majors_*.py` — contract / account_replay / paper_runtime / capital_sensitivity / pipeline。  
CLI：`bootstrap-server --mode majors` · `server-handoff` · `majors-hourly` · `majors-replay` · `admit-majors` · `paper-cycle-majors` · `majors-capital-sensitivity` · `run-majors` · `watch-majors` · `majors-preflight` · `majors-readiness` · `majors-refresh-15m` · `demo-checklist` · `ops-summary` · `clear-halt`（默认无交易所下单；密钥仅服务器 agent）。

### 阶段 2 — 策略与本地 paper 毕业（仍不上交易所）

- 固定规则不改参  
- 毕业门槛（代码：`prod/graduation.py`）：成交 ≥20 **或** completed_cycle ≥30；未 halt；`places_exchange_orders=false`；`live_allowed=false`  
- 决策枚举：`graduated_local` / `not_yet` / `blocked`（`python -m prod.cli status`）  
- `graduated_local` 仅本地毕业；**不**打开模拟盘/实盘；非 BTC/ETH 宇宙不可 `ready_for_demo_stage_consideration`  
- 回测与本地 paper 结论一致（同规则指纹）  

**出口：** 机器可读本地毕业通过 + 书面「允许进入交易所模拟盘」清单。**尚未**开模拟盘主循环。

### 阶段 3 — 交易所模拟盘（后期，**仅服务器**）

- 仅 BTC/ETH；密钥由**服务器 agent** 注入，本机不跑  
- 接 `runner` / OKX sandbox 策略主循环（服务器）  
- **必须在模拟盘看到可接受效果** 才进入阶段 4  
- 本机只维护：`demo-checklist` 工程就绪、部署文档、规则指纹  

**出口：** 服务器模拟盘达标；人工确认可申请实盘。

### 阶段 4 — 小资金实盘（最后，**仅服务器**）

- 硬顶起步 10U 级；密钥仅服务器 agent  
- 人工确认；连续监控  
- 禁止：补仓、加杠杆救回撤、改参  

**出口：** 实盘自动运行稳定；再谈扩容。

### 阶段 5 — 研究工厂降级

- audit 迁 `research/` 或 README 标明「非主链」  
- 新研究必须：BTC/ETH 或固定全集、10U 默认、如何挂 `prod`
