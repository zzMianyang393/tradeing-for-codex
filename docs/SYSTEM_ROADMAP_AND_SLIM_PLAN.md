### 加权总进度（主观）

**约 35%–40% → 100%「可自用自动交易系统」。**

- 若只论「研究工厂」：约 **70%+**（工具齐，结论多为负）  
- 若论「有可上盘策略」：约 **10%–20%**（无获批 edge）  
- 若论「执行系统骨架」：约 **40%**  

**最大缺口不是再写 100 个 audit，而是：生产主链 + 可交易策略宇宙 + 交易所自动执行闭环。**

┌─────────────────────────────────────────────────────────┐

│     prod/ · runner · exchange · executor · risk · state │
│     ten_u 信号引擎 · 未来 multi 策略插件                 │

- OKX 公共数据下载；10U 专用 1H + funding

- 仓库：研究代码与生产代码物理/文档隔离仍弱  
- **可验证正期望策略**（系统目标外的 alpha 问题）

## 4. 后续方向（按阶段，服务「自动交易系统」）

### 阶段 0 — 瘦身与冻结边界（0.5–1 天）

- [ ] 外置/删除 `reports/candidate_pool`（~5GB）  
- [ ] 外置可选 `data/basis`、`calendar_spread*`  
- [ ] `git rm --cached` data/reports；push 瘦代码  
- [ ] 文档声明：日常入口只有 `prod/` + `runner.py`

**出口：** 仓库 < 百 MB 级可推；本地仍可研究。

### 阶段 1 — 生产主链 MVP（1–2 周）**【主线】**

目标：`信号 → 风控 → 下单 → 同步 → 状态` 每小时自动跑。

1. **数据刷新守护**  
   - 10U：刷新 RAVE/LAB/ETH 1H + funding（实盘公共行情）  
   - 多币：增量 15m（可选）  
2. **Paper 主循环产品化**  
   - 定时 `paper-cycle`  
   - 开平仓、权益、halt、日志  
3. **执行适配双模式**  
   - `local_paper`：任意币（含 RAVE/LAB）  
   - `okx_demo`：仅 demo 支持的合约（ETH/BTC…）做**执行演练**  
   - `okx_live_capped`：默认关  
4. **宇宙矩阵检查**  
   - 脚本输出：live / demo 对每个 instId 是否可交易  

**出口：** 无人值守本地 paper ≥7 天；demo 上 ETH 能自动开平一笔。

### 阶段 2 — 模拟/本地验证毕业（2–4 周）

- 固定规则不改参  
- 毕业门槛（建议）：成交 ≥20 或运行 ≥30 天；无对账事故；回撤/停机规则触发正确  
- 10U：以 **local_paper 为准**（因 demo 无 RAVE/LAB）  
- 执行质量：以 **demo ETH** 为准  

**出口：** 书面「可进入小实盘」清单勾选完成。

### 阶段 3 — 小资金实盘（谨慎）

- 隔离 10U 袖套；硬顶；仅 live 有的合约  
- 人工确认开启；连续监控  
- 禁止：补仓、加杠杆救回撤、改参  

**出口：** 实盘自动运行稳定；再谈扩容。

### 阶段 4 — 多币生产轨（并行，不挡 10U）

- 从现有研究里挑 **demo/live 都有** 的币与低频规则  
- 同一套 admission → registry → runtime  
- 组合层后置  

### 阶段 5 — 研究工厂降级

- audit 迁 `research/` 或保持根目录但 README 标明「非主链」  
- 新研究必须回答：如何进入 `prod` 插件接口
