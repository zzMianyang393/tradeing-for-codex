# 新策略候选提案

更新时间：2026-07-10

## 候选 1：跨币种相对强弱轮动 (Cross-Coin Relative Strength Rotation)

### 利润来源假设

山寨币相对于 BTC 的强弱存在动量延续效应。当某币种相对 BTC 持续走强时，倾向于继续走强；反之亦然。

### 有效行情

- 牛市中后期：资金从 BTC 流向山寨币，强势山寨币持续跑赢
- 熊市初期：弱势山寨币加速下跌
- 不适合：普涨/普跌行情（相关性太高）

### 已有策略未覆盖的原因

当前所有策略都是单币种绝对指标（RSI、EMA、布林带等），没有跨币种相对强弱信号。

### 需要的数据

- 29 个币种的 15m K 线（已有）
- 计算每个币种相对 BTC 的 21 天收益率

### 无未来泄漏回测可行性

✅ 可以用当前 15m 数据完成。计算相对收益率时只使用历史数据。

### 可能的失效机制

- 市场相关性突然升高（如暴跌时所有币种同跌）
- 强势币种突然反转（动量反转）
- 交易成本侵蚀小收益

### 实现方式

```python
# 计算相对强弱
relative_strength = (coin_return_21d - btc_return_21d)

# 做多：相对强弱排名前 20% 且相对强弱 > 0
# 做空：相对强弱排名后 20% 且相对强弱 < 0
```

## 候选 2：多周期趋势确认 (Multi-Timeframe Trend Confirmation)

### 利润来源假设

当 4h 级别趋势明确时，15m 级别的顺势入场信号更可靠。大周期确定方向，小周期找入场点。

### 有效行情

- 明确的 4h 级别趋势（上涨或下跌）
- 不适合：4h 级别震荡

### 已有策略未覆盖的原因

当前策略只用 15m 数据判断趋势，没有大周期确认。大周期趋势可以过滤掉很多 15m 级别的假信号。

### 需要的数据

- BTC/ETH 的 4h K 线（已有：`data/BTC_4h.csv`, `data/ETH_4h.csv`）
- 其他币种的 4h 数据缺失，需要用 15m 重采样

### 无未来泄漏回测可行性

⚠️ 部分可行。BTC/ETH 有 4h 数据，其他币种需要从 15m 重采样，但重采样时不能使用未来数据。

### 可能的失效机制

- 4h 趋势反转时 15m 信号已经入场
- 重采样的 4h 数据质量不如原生 4h 数据
- 只有 BTC/ETH 有原生 4h 数据，其他币种可能有偏差

### 实现方式

```python
# 4h 趋势判断（用 BTC 作为基准）
btc_4h_trend = "up" if btc_ema20_4h > btc_ema50_4h else "down"

# 15m 入场（只在 4h 趋势方向入场）
if btc_4h_trend == "up":
    # 只做多，不做空
    signal = long_signal_for(symbol, bars, idx)
elif btc_4h_trend == "down":
    # 只做空，不做多
    signal = short_signal_for(symbol, bars, idx)
```

## 候选 3：波动率压缩突破 (Volatility Compression Breakout)

### 利润来源假设

当市场波动率压缩到极低水平后，往往会出现大幅突破。这种突破有方向性动量。

### 有效行情

- 长期横盘后的突破
- 不适合：已经处于高波动状态的市场

### 已有策略未覆盖的原因

当前 transition_breakout_long 只在 transition regime 触发，但没有专门检测波动率压缩。波动率压缩是一个独立的信号源。

### 需要的数据

- 15m K 线的 ATR 数据（已有）

### 无未来泄漏回测可行性

✅ 可以用当前 15m 数据完成。ATR 计算只使用历史数据。

### 可能的失效机制

- 假突破（突破后快速回撤）
- 压缩时间过长导致信号延迟
- 突破方向判断错误

### 实现方式

```python
# 检测波动率压缩
atr_20 = atr_20_period
atr_50 = atr_50_period
compression_ratio = atr_20 / atr_50

# 压缩条件：当前 ATR 明显低于历史水平
if compression_ratio < 0.7:
    # 等待突破
    if close > donchian_high:
        signal = long_signal
    elif close < donchian_low:
        signal = short_signal
```

| 优先级 | 候选 | 理由 |
|--------|------|------|
| 1 | 跨币种相对强弱轮动 | 逻辑独立，数据可用，无未来泄漏 |

| 3 | 波动率压缩突破 | 与 transition_breakout 有重叠风险 |
